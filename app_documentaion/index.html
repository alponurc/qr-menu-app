1. Folder structure

  • dinehub > public: Contains the public files.
  • dinehub > src > app: Contains the main application files.
  • dinehub > src > components: Contains reusable UI components.
  • dinehub > src > config: Contains configuration files.
  • dinehub > src > data: Contains data utilities.
  • dinehub > src > dish: Contains dish components.
  • dinehub > src > hooks: Contains custom hooks.
  • dinehub > src > items: Contains item components for different items.
  • dinehub > src > routes: Contains routes for the application.
  • dinehub > src > scss: Contains SCSS files for styling the application.
  • dinehub > src > stores: Contains Zustand stores for managing global state.
  • dinehub > src > svg: Contains SVG icons.
  • dinehub > src > types: Contains TypeScript types.

2. Install environment

Your computer should have Node.js and npm installed to manage the dependencies. You can download Node.js (which includes npm) from the following link: Node.js

3. Install dependencies

To install the project dependencies, run the following command in the project directory: npm install

4. Run the project

To run the project, use the following command: npm run dev

5. Build the project

To build the project, use the following command: npm run build
You can find more information on Next.js production builds in the Next.js documentation: Next.js Production Builds

6. Deploy the project

You can deploy managed Next.js with Vercel , or self-host on a Node.js server, Docker image, or even static HTML files.
You can find more information on Next.js deployment in the Next.js documentation: Next.js Deployment
